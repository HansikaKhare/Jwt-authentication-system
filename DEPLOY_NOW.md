## Part 1: Deploy Backend (5 minutes)

### Create Backend Service

1. **Click "New +" → "Web Service"**

2. **Connect Repository:**
   - Find: `HansikaKhare/Jwt-authentication-system`
   - Click **"Connect"**

3. **Configure Service:**
   ```
   Name: jwt-auth-backend
   Region: Oregon (default)
   Branch: main
   Root Directory: backend
   Runtime: Java
   Build Command: ./mvnw clean package -DskipTests
   Start Command: java -jar target/jwt-auth-system-1.0.0.jar
   ```

4. **Choose Plan:** Select **"Free"**

5. **Add Environment Variables:**
   Click **"Advanced"** tab, then add these:
   ```
   Key: SPRING_PROFILES_ACTIVE
   Value: production
   
   Key: JWT_SECRET
   Value: MySecretKey2024_VeryLongAndSecure_RandomString123!
   
   Key: PORT
   Value: 10000
   
   Key: JAVA_VERSION
   Value: 24
   ```

6. **Click "Create Web Service"**

7. **Wait for deployment** (3-5 minutes)
   - You'll see logs scrolling
   - Wait for: "Deployment complete"
   - Copy your URL: `https://jwt-auth-backend-xxxx.onrender.com`

---

## Part 2: Deploy Frontend (3 minutes)

### Create Static Site

1. **Click "New +" → "Static Site"**

2. **Connect Same Repository:**
   - Find: `HansikaKhare/Jwt-authentication-system`
   - Click **"Connect"**

3. **Configure Site:**
   ```
   Name: jwt-auth-frontend
   Region: Oregon (same as backend)
   Branch: main
   Root Directory: frontend
   Build Command: echo "Deploying static files..."
   Publish Directory: .
   ```

4. **Choose Plan:** Select **"Free"**

5. **Add Environment Variable:**
   ```
   Key: API_BASE_URL
   Value: https://jwt-auth-backend-xxxx.onrender.com/api
   (Replace xxxx with your actual backend URL from Part 1)
   ```

6. **Click "Create Static Site"**

7. **Wait for deployment** (1-2 minutes)

## 🆘 Troubleshooting

### Backend Won't Deploy

**Check Logs:**
1. Go to Render Dashboard
2. Click on `jwt-auth-backend`
3. Click **"Logs"** tab
4. Look for error messages

**Common Fixes:**
- Build fails → Check build command is correct
- Port error → Ensure PORT=10000 is set
- Database error → Render auto-creates PostgreSQL

### Frontend Shows "Cannot connect to backend"

**Solutions:**
1. Verify backend is running (check logs)
2. Check API_BASE_URL environment variable matches your backend URL exactly
3. Test backend directly: Visit `https://jwt-auth-backend-xxxx.onrender.com/api/auth/login` in browser

### CORS Error in Browser Console

**Fix:** Update backend CORS settings (already configured in your code!)

## 🎯 What Happens During Deployment?

### Backend Deployment:
1. Render clones your GitHub repo
2. Runs Maven build: `./mvnw clean package`
3. Creates JAR file
4. Provisions PostgreSQL database
5. Starts Spring Boot application
6. Assigns public URL

### Frontend Deployment:
1. Clones your repo
2. Copies all files from `frontend/` folder
3. Serves as static site
4. Assigns public URL
5. Injects environment variables
